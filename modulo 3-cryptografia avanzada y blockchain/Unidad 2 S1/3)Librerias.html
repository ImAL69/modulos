Tema 2.1: Implementación con Librerías

Buenas prácticas de desarrollo seguro.

Usa Estándares, No Inventos

La regla de oro de la criptografía aplicada es: Nunca escribas tu propio algoritmo criptográfico. Usa librerías auditadas y mantenidas por la comunidad.

  • Python: cryptography, PyCryptodome.
  • Java: Bouncy Castle.
  • Node.js: Módulo nativo crypto.
Configuración Segura

No basta con usar la librería, hay que configurarla bien. Errores comunes:

  • Usar modos de cifrado inseguros (como ECB). Usa siempre CBC o GCM.
  • Usar vectores de inicialización (IV) estáticos o nulos. El IV debe ser aleatorio para cada mensaje.
# MAL: IV estático (Inseguro)
iv = b'0000000000000000'

# BIEN: IV aleatorio (Seguro)
iv = os.urandom(16)
PROHIBIDO: Implementar AES o RSA "a mano" copiando fórmulas matemáticas. Un error de un bit en la implementación hace que el sistema sea vulnerable.
Caja de herramientas (qué usar para qué)
  • AES‑GCM / ChaCha20‑Poly1305 (AEAD): cifrado moderno con autenticación (evita “cifrar sin integridad”).
  • SHA‑256 / SHA‑3: integridad (huella).
  • HMAC‑SHA256: integridad + autenticidad en canales donde compartes una clave.
  • RSA/ECC: intercambio de claves y firma digital.

Consejo: si no necesitas recuperar el dato, no cifres: usa hash (ej. contraseñas).

Modo recomendado: AEAD (GCM / ChaCha20‑Poly1305)

AEAD significa “cifrado autenticado”: protege confidencialidad + integridad en un solo paso. En la práctica te evita el error clásico: “cifré el archivo, pero lo pudieron modificar”.

# Pseudoflujo seguro (AEAD)
nonce = os.urandom(12)
ciphertext, tag = encrypt(key, nonce, plaintext, aad=b"metadata")
plaintext = decrypt(key, nonce, ciphertext, tag, aad=b"metadata")

AAD (datos asociados) es metadata que no se cifra, pero queda protegida contra cambios.

Evidencia mínima de laboratorio
  1. Generación de clave (sin mostrarla completa en capturas públicas).
  2. Cifrado: archivo/mensaje → salida.
  3. Descifrado: salida → original (y prueba de igualdad).
  4. Integridad: altera 1 byte y muestra que falla verificación.
PROHIBIDO: usar ECB, IV fijo, “mi propio algoritmo”, o guardar claves en el repositorio.
