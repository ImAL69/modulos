Reglas del Laboratorio

Ética y responsabilidad en el uso de criptografía.

1. Manejo de Claves Privadas

Nunca compartas ni subas a la plataforma claves privadas reales. En los laboratorios, usaremos claves de prueba generadas exclusivamente para el ejercicio. Subir una clave real es una violación grave de seguridad.

2. Prohibido Cifrar Sistemas Ajenos

El uso de ransomware o herramientas de cifrado malicioso está terminantemente prohibido. Las herramientas de cifrado se usarán solo para proteger datos propios o de prueba.

3. Uso de Redes de Prueba (Testnet)

Para prácticas de Blockchain, utilizaremos redes de prueba (Testnets) o simuladores locales. No se permite realizar transacciones con dinero real en la red principal durante las clases para evitar pérdidas financieras.

Compromiso Ético:
Entiendo que estas herramientas son poderosas y me comprometo a usarlas de manera responsable y ética.

4. No publiques datos sensibles

Evita subir capturas con llaves, seeds, tokens, credenciales, correos personales o datos reales. Si necesitas evidencia, usa datos ficticios o ofusca.

5. No inventes criptografía

En prácticas y proyectos, usa librerías estándar (OpenSSL, libsodium, cryptography). Implementar AES/RSA “a mano” suele introducir fallas graves.

6. Evita prácticas riesgosas con blockchain

Trabaja solo con redes de prueba/simuladores. No compartas wallets reales ni realices transacciones con dinero real durante el curso.

✅ Confirmación:
Si llegaste hasta aquí, ya sabes lo esencial: proteger claves, usar testnet y documentar con ética. No necesitas presionar ningún botón para “aceptar”.
