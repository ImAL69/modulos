Rúbrica y Checklist de Entrega

Dimensión Excelente Aceptable Insuficiente
Reproducibilidad (Técnica) El pipeline se ejecuta de inicio a fin sin errores manuales. Código limpio, modular, comentado y el repositorio está completo. El código funciona pero requiere ajustes menores para correr o está desordenado. El código no ejecuta, faltan archivos o librerías, o es un "script espagueti" incomprensible.
Calidad Analítica (RA1+RA2) Limpieza justificada con EDA profundo, modelos comparados con rigor estadístico y métricas interpretadas en contexto de negocio. Análisis correcto pero superficial. Decisiones de limpieza o modelo poco justificadas o estándar. Errores conceptuales graves (fuga de datos, métricas incorrectas, falta de limpieza).
Operacionalización (RA3) Diagrama de pipeline claro y detallado. Estrategia de monitoreo, KPIs y riesgos bien definidos y mitigados. Pipeline incompleto o monitoreo genérico. Se mencionan riesgos pero no hay plan claro. No hay propuesta de pipeline, ni monitoreo, ni gestión de riesgos. Solo modelo aislado.

✅ Checklist Final de Entrega

  • ¿Incluí el informe final completo (PDF) con todas las 6 secciones obligatorias?
  • ¿El archivo ZIP o Repositorio tiene el código, el README.md y el requirements.txt?
  • ¿Hay evidencia visual clara del EDA (gráficos, tablas de calidad)?
  • ¿Está justificada técnicamente la elección del modelo final sobre las alternativas?
  • ¿Documenté explícitamente los riesgos del proyecto y cómo planeo mitigarlos?
