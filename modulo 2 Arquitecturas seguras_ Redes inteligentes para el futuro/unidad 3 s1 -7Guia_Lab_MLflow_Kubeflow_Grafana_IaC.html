Guía de laboratorio — Unidad 3

Esta guía cubre recursos prácticos mínimos para el proyecto integrador: MLflow, Kubeflow, Grafana e Infraestructura como Código (IaC).

1) Flujo práctico recomendado (paso a paso)

  1. Definir caso de IA (objetivo, datos, métrica principal).
  2. Registrar experimento en MLflow (parámetros, métricas, artefactos).
  3. Diseñar pipeline básico en Kubeflow (ingesta → entrenamiento → evaluación).
  4. Configurar tablero de observabilidad en Grafana (latencia, errores, disponibilidad).
  5. Versionar despliegue con IaC YAML para reproducibilidad.

2) Plantillas YAML mínimas (IaC educativa)

Deployment base (Kubernetes):

apiVersion: apps/v1
kind: Deployment
metadata:
  name: modelo-inferencia
spec:
  replicas: 2
  selector:
    matchLabels:
      app: modelo-inferencia
  template:
    metadata:
      labels:
        app: modelo-inferencia
    spec:
      containers:
      - name: api-modelo
        image: ejemplo/modelo:1.0
        ports:
        - containerPort: 8080

Service base (Kubernetes):

apiVersion: v1
kind: Service
metadata:
  name: svc-modelo
spec:
  selector:
    app: modelo-inferencia
  ports:
  - protocol: TCP
    port: 80
    targetPort: 8080

3) Evidencias que debe entregar el estudiante

  • Captura de ejecución o bitácora de experimento en MLflow.
  • Diagrama simple del pipeline en Kubeflow.
  • Captura de dashboard en Grafana con al menos 3 métricas.
  • Archivo YAML comentado y justificación de controles de seguridad.

4) Recursos auditivos y lecturas especializadas

  • Podcast: The Cloudcast (episodios de Kubernetes/MLOps).
  • Podcast: Software Engineering Daily (arquitecturas distribuidas).
  • Lectura: documentación oficial MLflow y Kubeflow (secciones "getting started").
  • Lectura: guías de observabilidad con Prometheus/Grafana.
