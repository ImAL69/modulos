📘 Guía de Lectura: NIST SP 800-115

Technical Guide to Information Security Testing and Assessment

¡Bienvenido/a al mundo de las evaluaciones de seguridad! El documento NIST SP 800-115 es una guía técnica fundamental que establece las bases para planificar, ejecutar y analizar pruebas y evaluaciones de seguridad de la información. Es un recurso invaluable, pero su naturaleza técnica y exhaustiva (80 páginas) puede resultar abrumadora.

Esta guía de lectura está diseñada para ser tu hoja de ruta. Te indicaremos qué secciones son esenciales para comprender la metodología general, cuáles son clave para conocer las técnicas específicas, y qué partes son de consulta para aspectos operativos y avanzados.

El objetivo no es leerlo todo de una vez, sino utilizarlo de forma estratégica para construir un conocimiento sólido sobre cómo se realizan las evaluaciones de seguridad en el mundo real.

📋 Parte 2: Técnicas de Examen (Review Techniques)

Técnicas pasivas para obtener información sin interactuar directamente con los sistemas objetivo.

📑 Revisión de Documentación (3.1)
Analiza políticas, planes de seguridad y procedimientos para encontrar brechas y debilidades que puedan llevar a controles mal implementados.
🔎 Qué buscar: Políticas obsoletas o que no cubren tecnologías actuales.
📊 Revisión de Logs (3.2)
Verifica que se esté registrando la información correcta y que la organización cumpla con sus políticas de gestión de logs. Puede revelar accesos no autorizados o intrusiones.
📌 Tipos de logs: Autenticación, sistema, firewall, IDS, antivirus.
🔐 Revisión de Reglas (3.3)
Evalúa reglas en firewalls, ACLs de routers y firmas de IDS/IPS para asegurar que son necesarias, efectivas y no crean brechas de seguridad.
🛡️ Chequeo común: Verificar que no haya reglas innecesarias o puertos abiertos de más.
⚙️ Revisión de Configuración (3.4)
Identifica debilidades en la configuración de seguridad de sistemas, como servicios innecesarios, cuentas inseguras o configuraciones fuera de la política de hardening.
🖥️ Herramientas: SCAP para automatizar la comprobación.
👂 Network Sniffing (3.5)
Técnica pasiva que monitorea el tráfico de red para identificar protocolos inseguros (ej. telnet), actividades no autorizadas o información sensible sin cifrar (como contraseñas).
⚠️ Limitación: No funciona con tráfico cifrado.
📦 File Integrity Checking (3.6)
Identifica cambios en archivos del sistema mediante checksums, ayudando a detectar la modificación de archivos críticos o la presencia de herramientas de atacantes.
💡 Recomendación: Usar checksums criptográficos (SHA).
📌 RESUMEN (Tabla 3-1, Pág. 22): Las técnicas de revisión evalúan políticas, logs, reglas, configuraciones y tráfico de forma pasiva, con mínimo riesgo para los sistemas.

🎯 Parte 3: Identificación y Análisis de Objetivos

Técnicas activas para descubrir dispositivos, puertos, servicios y vulnerabilidades potenciales.

🌐 Network Discovery (4.1)
Descubre hosts activos en la red, ya sea de forma pasiva (escuchando) o activa (enviando paquetes como pings). Ayuda a identificar dispositivos no autorizados (rogue devices).
🔎 OS Fingerprinting: Determinar el SO de un host por sus respuestas a paquetes.
🚪 Port and Service ID (4.2)
Usa escáneres de puertos para identificar puertos abiertos y los servicios/aplicaciones que corren en ellos (ej. banner grabbing).
⚠️ Precauciones: Puede generar ruido en la red y alertar a IDS. Usar con cuidado.
🔬 Vulnerability Scanning (4.3)
Identifica hosts, sistemas operativos y aplicaciones, y los compara con bases de datos de vulnerabilidades conocidas para encontrar puntos débiles (ej. parches faltantes, malas configuraciones).
📉 Debilidad: Alta tasa de falsos positivos. Requiere interpretación humana.
📡 Wireless Scanning (4.4)
Identifica dispositivos inalámbricos (puntos de acceso, clientes) no autorizados, evalúa su configuración de seguridad y puede localizarlos físicamente.
📡 Modos: Pasivo (solo escucha) y Activo (intenta conectar y probar).
📌 RESUMEN (Tabla 4-1, Pág. 33): La identificación y análisis usa técnicas activas como escaneo de red y puertos para descubrir activos y vulnerabilidades superficiales.

⚔️ Parte 4: Validación de Vulnerabilidades

Técnicas de alto riesgo que buscan probar y confirmar la existencia de vulnerabilidades, demostrando el impacto real de una explotación.

🔑 Password Cracking (5.1)
Recupera contraseñas desde hashes (interceptados o copiados del sistema) mediante ataques de diccionario, híbridos, fuerza bruta o rainbow tables.
🎯 Objetivo: Identificar cuentas con contraseñas débiles y validar la política de contraseñas.
💣 Penetration Testing (5.2)
Simula ataques reales para explotar vulnerabilidades y ver qué nivel de acceso se puede conseguir. Busca combinaciones de vulnerabilidades para lograr un impacto mayor.
Fases: Planning → Discovery (Info gathering + vuln analysis) → Attack → Reporting. (Ver Figuras 5-1 y 5-2).
🗣️ Social Engineering (5.3)
Prueba el "factor humano" engañando a usuarios para que revelen información (ej. phishing). Evalúa la concienciación y el cumplimiento de procedimientos.
📧 Ejemplo: Enviar correos de phishing para ver quién cae y reporta.
⚠️ ALTO RIESGO: Estas técnicas pueden causar indisponibilidad de servicios o daños en sistemas. Requieren una planificación meticulosa, autorización explícita (ROE) y ejecución por personal altamente capacitado.
