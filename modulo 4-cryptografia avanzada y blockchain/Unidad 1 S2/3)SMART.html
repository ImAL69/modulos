Smart Contracts

Son protocolos informáticos autoejecutables que facilitan, verifican o hacen cumplir la negociación o ejecución de un contrato. El código se almacena en la blockchain y se ejecuta automáticamente cuando se cumplen las condiciones predefinidas ("If This Then That").

Principio clave: “el contrato se ejecuta, no interpreta”

Un smart contract no “entiende” intención. Si la lógica está mal, se ejecuta igual. Por eso, en el mundo real se recomienda:

  • Validaciones estrictas de entradas y estados.
  • Controles de acceso (roles, owners, multisig).
  • Auditoría del código y pruebas (unit tests) antes de desplegar.
Patrón mental: Estado + Reglas

Piensa el contrato como una máquina de estados:

Estado: {CREADO → ACTIVO → CERRADO}
Regla: "Solo el rol X puede pasar de ACTIVO a CERRADO"
Evento: "PagoRecibido()" o "EntregaConfirmada()"

Cuando el estado cambia, se generan eventos (logs) que pueden auditarse.

Fallas típicas (para tu análisis)
  • Reentrancy: una llamada externa vuelve a entrar antes de actualizar estado.
  • Overflow/underflow: aritmética fuera de rango (hoy mitigado en Solidity moderno).
  • Permisos débiles: cualquiera puede ejecutar funciones críticas.
  • Oráculo manipulable: dato externo no confiable (precio, clima, etc.).
Ejemplos de uso (bien planteados)
  • Escrow (depósito en garantía): fondos se liberan si se confirma entrega.
  • Subastas: reglas transparentes y verificables.
  • Certificados verificables: hash del diploma/acta registrado para verificación.
