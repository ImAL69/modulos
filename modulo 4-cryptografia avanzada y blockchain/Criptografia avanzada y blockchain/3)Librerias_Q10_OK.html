Buenas prácticas para desarrollo criptográfico seguro usando librerías estándar

📦 Regla #1: Usa Estándares, No Inventos

La regla fundamental en criptografía aplicada es: Nunca escribas tu propio algoritmo criptográfico. Los algoritmos son complejos y un error mínimo puede comprometer toda la seguridad. Usa exclusivamente librerías auditadas y mantenidas por la comunidad.

⚠️ Importante:

Incluso usando librerías estándar, debes configurarlas correctamente. Una librería segura mal configurada es tan peligrosa como un algoritmo inseguro.

• Configuración Segura: Evita Errores Comunes

No basta con usar la librería correcta; la configuración determina la seguridad real. Estos son los errores más frecuentes:

IV Estático o PredictiblePermite ataques de diccionario y análisis de patronesIV aleatorio único para cada cifrado

💡 Regla práctica:

Si no necesitas recuperar el dato original, no cifres — usa hash (ejemplo: contraseñas). El hashing es unidireccional y más seguro para verificación.

🔒 Modo Recomendado: AEAD (Cifrado Autenticado)

AEAD (Authenticated Encryption with Associated Data) protege confidencialidad + integridad en un solo paso. Evita el error clásico: "cifré el archivo, pero alguien pudo modificar el contenido".

# Ejemplo completo con AES-GCM (Python cryptography)

from

cryptography.hazmat.primitives.ciphers.aead

import

AESGCM

import

os

# 1. Generar clave y nonce

key = AESGCM.generate_key(bit_length=256)

# Clave de 256 bits

nonce = os.urandom(12)

# Nonce aleatorio (12 bytes para GCM)

# 2. Crear cifrador

aesgcm = AESGCM(key)

# 3. Cifrar con datos asociados (AAD)

plaintext =

b"Datos confidenciales del usuario"

associated_data =

b"metadata:version=1,user=123"

# No se cifra, pero se autentica

ciphertext = aesgcm.encrypt(nonce, plaintext, associated_data)

# 4. Descifrar y verificar

try

: decrypted = aesgcm.decrypt(nonce, ciphertext, associated_data) print(

AAD (Associated Data): Metadatos que no se cifran pero se protegen contra modificaciones. Útil para headers, timestamps, IDs que necesitan autenticidad pero no confidencialidad.

📋 Evidencia Mínima para Laboratorio/Reporte

Al implementar criptografía en tus proyectos, documenta estos elementos esenciales:

  1. Generación de clave segura: Muestra el método (sin revelar la clave completa en capturas públicas).
  2. Proceso de cifrado: Archivo/mensaje original → texto cifrado (mostrar transformación).
  3. Proceso de descifrado: Texto cifrado → original (y prueba de igualdad con hash).
  4. Verificación de integridad: Alterar 1 byte y demostrar que falla la verificación.
  5. Configuración: Mostrar parámetros usados (algoritmo, modo, tamaño de clave).

LISTA DE PROHIBIDOS:

• Usar ECB mode • IV fijo o predecible • "Mi propio algoritmo" • Guardar claves en el repositorio • Cifrar sin autenticación • Claves cortas (<128 bits) • Semillas aleatorias no criptográficas

🔐 Recuerda:

La seguridad criptográfica es una cadena: solo es tan fuerte como su eslabón más débil. Una configuración incorrecta anula incluso el algoritmo más fuerte.
