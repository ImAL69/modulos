Tema 2.2: Gestión de Claves y Secretos

El secreto de la seguridad está en proteger los secretos

La criptografía transforma el problema de proteger datos en el problema de proteger claves. Si un atacante obtiene tu clave, todo el cifrado es inútil. La gestión adecuada de claves es lo que separa sistemas seguros de aquellos que solo parecen seguros.

• Ciclo de Vida Completo de una Clave
1

🔑 Generación Segura

Debe hacerse con CSPRNG (Cryptographically Secure Pseudo-Random Number Generator). Nunca usar rand(), random() o funciones no criptográficas. Longitudes mínimas: AES-128 bits, RSA-2048 bits, ECC-256 bits.

2

🗄️ Almacenamiento Seguro

NUNCA en código fuente, nunca en repositorios. Usar variables de entorno, sistemas de gestión de secretos (KMS), o HSMs. Las claves deben estar cifradas en reposo y accesibles solo por aplicaciones autorizadas.

3

• Rotación Periódica

Cambiar claves regularmente limita el daño si una es comprometida. Frecuencias típicas: claves de sesión (diarias), claves de aplicación (30-90 días), claves maestras (1-2 años). Implementar sin interrupción del servicio.

4

• Revocación y Destrucción

Borrado seguro cuando la clave ya no es necesaria, ha expirado o ha sido comprometida. No solo eliminación lógica; sobrescribir almacenamiento físico. Mantener registro de claves destruidas para auditoría.

5

📋 Auditoría y Monitoreo

Registrar todo acceso a claves: quién, cuándo, para qué. Alertar sobre accesos inusuales o fuera de horario. Revisar periódicamente permisos y eliminar accesos innecesarios.

6

• Políticas y Gobernanza

Documentar políticas claras: quién puede generar claves, rotación obligatoria, procedimientos de recuperación. Separación de responsabilidades: diferentes personas para generación, almacenamiento y uso.

🛡️ Herramientas de Gestión de Secretos (KMS):

Usa servicios profesionales para manejar tus secretos. Estas herramientas proveen:

  • Cifrado automático en reposo y en tránsito
  • Control de acceso granular (IAM, RBAC)
  • Rotación automática de claves
  • Auditoría completa de todos los accesos
  • HSM integrado para máxima seguridad
⚠️

¡NUNCA subas secretos a GitHub/GitLab!

Usa .gitignore para archivos con claves y sistemas de secretos para entornos de producción.
• Sistemas KMS Recomendados
•
AWS Key Management Service
Integración nativa con servicios AWS. Soporte para CMKs (Customer Master Keys), envelope encryption, y rotación automática. Costo: ~$1/mes por clave.
•
Azure Key Vault
Soporte para claves, secretos y certificados. Integración con Azure Active Directory. Políticas de acceso granular y logging detallado.
•
HashiCorp Vault
Open-source y self-hosted. Dinámico secret generation, lease management, y políticas flexibles. Ideal para entornos híbridos o on-premise.
•
Google Cloud KMS
HSM como servicio, importación de claves externas. Integración con Cloud IAM y Cloud Audit Logs. Soporte para múltiples algoritmos.

• Política de Gestión de Claves (Modelo)

Una política efectiva debe incluir estos 6 puntos esenciales:

  1. Generación: Usar solo CSPRNG, longitudes mínimas según estándares, validar calidad de entropía.
  2. Almacenamiento: NUNCA en código, usar KMS/HSM, cifrado en reposo, acceso por identidad de servicio.
  3. Acceso: Principio de mínimo privilegio, autenticación fuerte, registro de todos los accesos.
  4. Rotación: Automática cada 90 días máximo, notificación antes de expiración, soporte para versiones.
  5. Revocación: Procedimiento inmediato por incidente, destrucción segura, lista de revocación.
  6. Auditoría: Revisión trimestral de accesos, verificación de políticas, reporte de cumplimiento.
